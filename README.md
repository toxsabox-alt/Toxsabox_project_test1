# Лендинг: Архитектор цифровой автоматизации

Готовый адаптивный лендинг с:
- продающим первым экраном;
- промо-анимацией (floating orb + animated pipeline progress);
- блоками услуг, кейсов и процесса работы;
- формой заявки на консультацию с валидацией;
- базовой логикой сохранения заявок в `localStorage`.

## Структура проекта

```text
.
├─ index.html     # Разметка всех секций лендинга
├─ styles.css     # Стили, адаптив, анимации
├─ script.js      # Reveal-анимации и обработка формы
└─ README.md      # Документация
```

## Быстрый запуск

### Вариант 1: просто открыть файл
1. Откройте `index.html` в браузере двойным кликом.

### Вариант 2: локальный сервер (рекомендуется)
Если установлен Node.js:
```bash
npx serve .
```
После запуска откройте адрес, который появится в терминале (обычно `http://localhost:3000`).

## Что уже реализовано

### 1) Продающая структура
- Hero-блок с ключевым оффером, CTA-кнопками и метриками;
- Блок "Услуги";
- Блок "Кейсы";
- Блок "Процесс";
- Блок "Заявка на консультацию".

### 2) Промо-анимации
- Плавающие световые сферы (`@keyframes float`);
- Анимированная полоса прогресса pipeline (`@keyframes pulseLine`);
- Плавное появление секций при скролле через `IntersectionObserver` (`.reveal`).

### 3) Форма заявки
Поля:
- имя;
- телефон;
- email;
- комментарий.

Валидация:
- имя: минимум 2 символа;
- телефон: минимум 10 цифр;
- email: проверка регулярным выражением.

Текущее поведение:
- при успешной отправке данные сохраняются в `localStorage` (ключ `leads`);
- пользователю показывается статус отправки.

## Подключение реальной отправки формы (backend/API)

Сейчас форма демонстрационная. Для production нужно заменить логику в `script.js` на запрос к вашему API.

Пример:

```js
async function submitLead(payload) {
  const response = await fetch("https://your-domain.ru/api/leads", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(payload),
  });

  if (!response.ok) {
    throw new Error("Ошибка отправки");
  }
}
```

И внутри `submit`-обработчика:
```js
await submitLead(payload);
setStatus("Заявка отправлена.", "success");
```

## Интеграция с Telegram (альтернатива)

Можно отправлять заявки в Telegram-бот:
1. Создать бота через BotFather;
2. Получить `BOT_TOKEN`;
3. Узнать `CHAT_ID`;
4. На backend сделать endpoint, который принимает форму и вызывает Telegram API:
   `https://api.telegram.org/bot<BOT_TOKEN>/sendMessage`.

Важно: не храните токен бота в frontend-коде.

## Кастомизация

### Контент
- Заголовки и тексты: `index.html`;
